# Field Guide

A compact starter notebook for ideas, research trails, and useful patterns that deserve a good home.

## The Shape

This repository treats Markdown as the source material and the generated site as the reading surface. Write notes in plain files, organize them with headings, and let the build script turn them into a navigable notebook.

Good notebooks usually have a few habits:

- One clear topic per file.
- Short sections with descriptive headings.
- Links back to source material.
- Small examples that make the idea easy to reuse.
- A few open questions so the next pass has a place to begin.

## A Tiny Research Loop

When a topic is fuzzy, use a repeatable loop instead of waiting for perfect clarity.

1. Capture the raw question.
2. Write the current best guess in one paragraph.
3. Add evidence, examples, or counterexamples.
4. Name what would change your mind.
5. Return later and tighten the note.

```text
question -> guess -> evidence -> revision -> next question
```

## Useful Note Types

| Type | Use it for | Nice ending |
|---|---|---|
| Map | Explaining a topic area | A short glossary |
| Log | Recording discoveries over time | A dated change note |
| Recipe | Repeating a process | A checklist |
| Decision | Choosing between options | Tradeoffs and review date |
